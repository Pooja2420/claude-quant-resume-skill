# Contributing

Thanks for helping improve the quant resume writer skill.

## What to contribute

- **Company signals** — JD patterns and ATS platform quirks for firms not yet covered
- **Role coverage** — risk quant, desk quant, quant structurer, actuarial
- **International ATS** — EU/UK (Europass patterns), Singapore, Hong Kong, Canada
- **Keyword lists** — new tools/frameworks entering quant stacks (e.g., Rust, JAX, Triton)
- **Bug fixes** — wrong ATS platform listed, outdated keyword, broken formatting rule

## Ground rules

1. **No fabrication** — never add "common" keywords that aren't verified from real JDs
2. **Cite your source** — if adding company-specific signals, note where they come from (anonymized is fine: "from 3 real JDs seen in Jan 2026")
3. **Keep it concise** — the skill is read by a language model token by token; every line costs
4. **Test it** — run a sample resume through Claude with the updated skill before submitting

## How to submit

1. Fork the repo
2. Create a branch: `git checkout -b improve/company-signals` or `fix/keyword-list`
3. Make your changes
4. Open a PR with a clear description of what changed and why
